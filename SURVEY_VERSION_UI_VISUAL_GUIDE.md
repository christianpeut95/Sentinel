## ?? Animation & Behavior

### On Page Load
```
1. Page renders
2. Version History section appears
3. Versions load from database
4. Current version highlighted
5. Action buttons enabled/disabled based on status
```

### On Publish Click
```
1. User clicks [?? Publish]
2. Confirmation dialog appears
3. User confirms
4. Button shows spinner: [?]
5. AJAX call to API
6. Success: Page reloads
7. New Active version shown
8. Previous Active ? Archived
```

### On Archive Click
```
1. User clicks [?? Archive]
2. Confirmation dialog appears
3. User confirms
4. Button shows spinner: [?]
5. AJAX call to API
6. Success: Page reloads
7. Version status ? Archived
8. Action buttons removed
```

### On View Click
```
1. User clicks [??? View]
2. Browser navigates to version page
3. URL: /Settings/Surveys/SurveyTemplateDetails?id={versionId}
4. New page loads with that version's data
5. "Current" indicator shows which version
```

## ? Visual Checklist

When viewing version history, you should see:

**Details Page:**
- [ ] Blue header with "Version History" title
- [ ] Badge showing total number of versions
- [ ] Scrollable list (if many versions)
- [ ] Current version highlighted in blue
- [ ] Status badge for each version
- [ ] Version notes displayed
- [ ] Created/Published dates
- [ ] Creator/Publisher names
- [ ] Action buttons (View, Publish, Archive)
- [ ] Buttons disabled appropriately

**Edit Page:**
- [ ] Data table with all versions
- [ ] Column headers clearly labeled
- [ ] Active version row is green
- [ ] Current version has "Current" badge
- [ ] Publish button only on drafts
- [ ] Info alert explaining versioning
- [ ] All dates formatted consistently
- [ ] Actions column with appropriate buttons

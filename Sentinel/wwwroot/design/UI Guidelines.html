01 · Identity
Brand
Sentinel is professional, quietly confident, and built for the people who stay up on the late shifts. The identity is clean, data-forward, and never alarmist — even when the data is alarming.
Mission & Voice
Professional, not clinical

We work in healthcare but we don't talk like a hospital manual. Clear sentences, no jargon — the data does the talking.

Watchful, not alarmist

Signal Green means healthy. We reserve red for genuine escalation. The system earns trust by being calm when things are fine.

Precise, not terse

Every label, tooltip, and empty state is written with care. Precision matters when a number can mean a life.

Confident, not corporate

We don't pad copy with qualifiers. We state what we know, flag what we don't, and always give the user a next step.

08 · Components
Buttons
Four variants covering the full intent range. Use Primary for default actions, Signal for constructive confirmations, Ghost for secondary actions, and Danger for destructive ones.
Variants
Sizes
Usage
✓ Do
Use Signal for the primary call-to-action when the action is constructive — confirming a case, submitting a report, creating a contact.
✕ Don't
Stack two filled buttons side-by-side at the same visual weight. One action should clearly be primary.
✓ Do
Use Ghost for cancel, dismiss, or any secondary option that appears alongside a filled button.
✕ Don't
Use Danger for anything reversible. Reserve it only for permanent or high-consequence actions like "Delete case".

16 · Guidelines
Voice & Tone
Precision matters when a number can mean a life. Every word in the UI earns its place.
State what you know

Don't hedge unnecessarily. "14 contacts identified" not "approximately 14 possible contacts may have been identified".

Flag what you don't

When data is incomplete or unverified, say so plainly. "Awaiting lab confirmation" is better than silence or an asterisk.

Give a next step

Empty states, errors, and confirmations always include one clear action. Never leave the user stranded.

Prefer active voice

"Contact Amara Osei" not "A contact should be made with Amara Osei". The system talks to a person doing a job.

Copy Examples
Context✕ Don't write✓ Write instead
Button label Submit form Save case
Empty state No data found. No cases match your filters. Try broadening the date range.
Error An error occurred. Couldn't save the case. Check your connection and try again.
Confirm delete Are you sure? Delete Case 2026-0427-A? This cannot be undone.
17 · Guidelines
Do's & Don'ts
Common mistakes and how to avoid them.
✓ Use mesh backgrounds for hero moments
Login screens, empty states, and marketing pages benefit from the mesh. It signals "you are entering a space".
✕ Never mesh behind data
Tables, forms, and charts must always sit on Bone, Paper, or Chalk. Textured backgrounds destroy data legibility.
✓ Use Mono for all numeric data
Case counts, dates, percentages, and IDs should always render in Geist Mono so numbers align in tables.
✕ Don't use Lime as a primary action colour
Lime is a tertiary pop colour for data visualisation accents. It's not suitable for buttons or status indicators.
✓ Respect the signal: green = healthy
Signal Green means "good news". Don't use it to flag a warning or neutral state — it will undermine trust.
✕ Don't stack more than 2 nav levels
Sentinel's users are on duty, under time pressure. Deep nav hierarchies are a UX failure.
✓ Keep the mark clear
Never recolour, stretch, rotate, or add effects to the Sentinel mark. Use the approved lockups only.
✕ Don't invent new status colours
The four status colours (Outbreak, Watch, Resolved, Info) are the complete set. Adding custom colours breaks the system's semantic trust.
